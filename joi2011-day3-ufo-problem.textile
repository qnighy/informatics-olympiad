<strong>出力のみの課題 (Output-only task)</strong>

IOI 星の宇宙人は日本にある JOI 公園 (ただし，一般の人には存在は秘密にされている) に UFO の飛行場を作る計画を立てている．IOI 星の宇宙人は JOI 公園を調査し，UFO が着陸可能な場所を示した地図を作成した．

JOI 公園は東西 <var>W</var> メートル，南北 <var>H</var> メートルの長方形であり，<var>1</var> メートル四方の正方形の区画に分けられている．区画は全部で <var>W \times H</var> 個あり，西から <var>x</var> 列目，北から <var>y</var>f 行目の区画を <var>(x, y)</var> と表す．北西の角の区画は <var>(1, 1)</var> であり，南東の角の区画は <var>(W, H)</var> である．各区画は「着陸可能」か「着陸不可能」かのいずれかであり，「着陸可能」は “.”，「着陸不可能」は “w” で表される．

IOI 星の宇宙人が作る UFO は幅 <var>B</var> メートル，奥行き <var>D</var> メートルにおさまる．この UFO は仕様上，設計図での上下左右がそれぞれ UFO において北，南，西，東の方角となるように，また設計図での左上の角がJOI 公園のいずれかの区画の左上と一致するように着陸しなければならない．UFO の設計図は <var>D</var> 行 <var>B</var> 列のマス目からなる．マスは全部で <var>B \times D</var> 個あり，左から <var>i</var> 列目，上から <var>j</var> 行目のマスを <var>(i, j)</var> と表す．左上のマスは <var>(1, 1)</var> であり，右下のマスは <var>(B, D)</var> である．マス <var>(i, j)</var> は UFO の西の端から <var>i - 1</var> メートル ～ <var>i</var> メートル，北の端から <var>j - 1</var> メートル ～ <var>j</var> メートルの正方形の範囲に UFO の一部が含まれるか否かを示す．各マスは UFO の一部が「含まれる」か「含まれない」かのいずれかであり，含まれる場合は “O” (大文字のオー)，含まれない場合は “.” で表される．

ある <var>1</var> 個の UFO が着陸する時，設計図上のマスで UFO に「含まれる」全ての場所は，着陸時に「着陸可能」な区画に含まれなければならない．また複数個の UFO が着陸する時は，ある UFO の設計図上のマスで「含まれる」全ての場所は，別の UFO の設計図上のマスで「含まれる」いずれの場所とも辺を共有することはできない．ただし，UFO は角を共有する場所に着陸することはできる．

あなたは奇しくも IOI 星の親善大使として選ばれた．そこで，あなたには，出来る限り多くの UFO の着陸が可能な飛行場を作る計画書を作って欲しい．

h2. 課題

JOI 公園に作る UFO の飛行場の計画書を作成せよ．より多くの個数の UFO を着陸させる計画書を作ることにより，あなたはより高い得点を得る．計画書は JOI 公園の地図上に着陸した UFO を書き加えたものであり，JOI 公園の「着陸可能」な区画のうち，着陸した UFO が含まれる区画を “O” (大文字のオー)，着陸した UFO が含まれない区画を “.” で表し，「着陸不可能」な区画は “w” で表す．

h2. 制限

| <var>1 \leq B \leq 5</var> | UFO の幅 (メートル) |
| <var>1 \leq D \leq 5</var> | UFO の奥行き (メートル) |
| <var>1 \leq W \leq 200</var> | JOI 公園の東西方向の大きさ (メートル) |
| <var>1 \leq H \leq 200</var> | JOI 公園の南北方向の大きさ (メートル) |

h2. 入力

入力ファイルは以下の形式で与えられる．
* <var>1</var> 行目には整数 <var>B, D</var> が空白を区切りとして書かれており，UFO の幅が <var>B</var> メートル，奥行きが <var>D</var> メートルであることを表す．
* 続く <var>D</var> 行には UFO の設計図の情報が書かれている．<var>j + 1</var> 行目 <var>(1 \leq j \leq D)</var> には，<var>B</var> 文字の文字列が書かれており，<var>i</var> 文字目 <var>(1 \leq i \leq B)</var> には設計図のマス <var>(i, j)</var> を表す “O” または “.” の文字が書かれている．
* <var>D + 2</var> 行目には整数 <var>W, H</var> が空白を区切りとして書かれており，JOI 公園の東西方向の大きさが <var>W</var> メートル，南北方向の大きさが <var>H</var> メートルであることを表す．
* 続く <var>H</var> 行には JOI 公園の地図の情報が書かれている．<var>y + D + 2</var> 行目 <var>(1 \leq y \leq H)</var> には，<var>W</var> 文字の文字列が書かれており，<var>x</var> 文字目 <var>(1 \leq x \leq W)</var> には公園の区画 <var>(x, y)</var> を表す “.” または “w” の文字が書かれている．

h2. 出力

出力の <var>y</var> 行目 <var>(1 \leq y \leq H)</var> には公園の北から <var>y</var> 行目の状態を表す <var>W</var> 文字の文字列を書け．<var>y</var> 行目 <var>(1 \leq y \leq H)</var> の <var>x</var> 文字目 <var>(1 \leq x \leq W)</var> は区画 <var>(x, y)</var> の状態を表す “O” または “.” または “w” のいずれかの文字である．

h2. 提出方法

各入力データに対する出力を提出せよ．その際，出力のセクションで指定された形式に一致するかのみがチェックされる．

h2. 採点基準

各入力ファイルには以下に示す <var>X</var> の値が割り当てられている．参加者が提出した計画書のうちの着陸できる UFO の個数の最大値を <var>Y</var> とする．あなたの計画書が問題の条件を満たさない場合，あなたの得点は <var>0</var> 点である．条件を満たす場合，<var>α</var> をあなたの計画書における着陸できる UFO の個数とし，

* <var>α \leq X</var> の時は，<var>\frac{4α}{X}</var> の小数第 <var>2</var> 位を四捨五入した値，
* <var>X &lt; α</var> の時は，<var>4 + 16 \times ( \frac{α-X}{Y-X} ) ^2</var> の小数第 <var>2</var> 位を四捨五入した値，

があなたの得点となる．
